## 说明

本文约定如下：

- `NEXT_PUBLIC_` 开头的变量会暴露到前端
- 不带 `NEXT_PUBLIC_` 的变量默认按服务端敏感配置处理
- `YOURS-`、`XXX`、空值都表示模板占位，不是可直接使用的真实值

## 不敏感配置

<Callout title="不敏感配置" type="info">
  这一类通常是公开配置或前端可见配置，主要用于站点展示、页面行为、路由、价格展示和非敏感业务参数。凡是 `NEXT_PUBLIC_` 开头的变量，都默认视为会暴露到客户端。
</Callout>

这组变量用于定义站点名称、基础地址、代码仓库地址以及一些全局展示参数

### 用途说明

| 变量名 | 说明 |
| --- | --- |
| `NEXT_PUBLIC_APP_NAME` | 站点名称 |
| `NEXT_PUBLIC_BASE_URL` | 站点主域名，本地一般是 `http://localhost:3000` |

| `NEXT_PUBLIC_GITHUB` | 仓库主页地址 |
| `NEXT_PUBLIC_GITHUB_BASE_URL` | 仓库默认分支根地址，通常用于拼接源码链接 |
| `NEXT_PUBLIC_STYLE_ICON_COLOR` | UI 图标主色 |
| `NEXT_PUBLIC_STYLE_SVG_ICON_SIZE` | SVG 图标尺寸 |
| `NEXT_PUBLIC_STYLE_WATERMARK_TEXT` | 页面水印文本 |

## 敏感配置

<Callout title="敏感配置" type="warn">
  这一类通常包含私钥、数据库连接、Webhook 密钥、服务端令牌和内部调试开关。不要把这类变量暴露到前端，也不要把真实值提交到仓库。

## 数据库配置

项目注释里已经强调了数据库权限隔离的重要性这里建议把生产与本地环境严格分开，并避免把高权限连接串直接用于日常开发

<TrophyCard title="结合上述流程, 从用户视角看, FumaDocs的扩展口" icon={<DatabaseZapIcon />}>
  - 本地: `DATABASE_URL="postgresql://YOURS_app:YOURS_PASSWORD@localhost:5432/postgres?sslmode=disable&schema=YOURS"`
  - 线上事务模式(推荐): `DATABASE_URL="postgresql://YOURS_app.PROJECT_ID:PASSWORD@SERVER_HOST.pooler.supabase.com:6543/postgres?sslmode=require&schema=YOURS&pgbouncer=true"`
  - 线上会话模式: `DATABASE_URL="postgresql://YOURS_app.PROJECT_ID:PASSWORD@SERVER_HOST.pooler.supabase.com:5432/postgres?sslmode=require&schema=YOURS"`
</TrophyCard>

### 用途说明

| 变量名 | 说明 |
| --- | --- |
| `DATABASE_URL` | Prisma 和后端服务的数据库连接串 |

### 配置注意事项

- 生产环境优先使用项目专属账号，不要直接使用超级用户
- 模板明确提示不要随意使用 `sslmode=require/no-verify` 的不安全变种
- 本地环境因为权限与证书条件不同，通常会禁用 SSL

## 建议的分层理解

如果只从运维和排查角度理解，可以把这些变量分成四层：

- `公开展示层`: 站点名称、URL、主题、水印、前端开关
- `业务配置层`: 套餐价格、积分规则、支付提供商、模型名、FAQ 行为
- `基础设施层`: 数据库、Clerk、Stripe、OpenRouter、Upstash
- `调试与模拟层`: mock、debug、隐藏开关、缓存调试

## 使用建议

- 本地开发先最小化配置：`DATABASE_URL`、Clerk、Stripe、OpenRouter 至少要能跑通核心流程
- 所有 `NEXT_PUBLIC_` 变量都要默认视为可被用户看到，不要放入任何私密信息
- 支付、Webhook、数据库、第三方服务私钥建议按环境分开管理，不要共用
- mock 和 debug 开关在生产环境上线前应逐项复核
